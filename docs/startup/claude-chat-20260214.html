  2. Key Technical Concepts:                                                                                            
     - **SPL (Structured Prompt Language)**: SQL-inspired declarative LLM orchestration — Lexer → Parser → Analyzer     
   → Optimizer → Executor pipeline                                                                                      
     - **CTEClause.nested_prompt**: New field added to store a full `PromptStatement` inside a CTE definition           
     - **ExecutionStep.cte_stmt**: New field to carry the nested PromptStatement to the executor                        
     - **Parallel CTE execution**: `asyncio.gather()` to run independent CTEs concurrently, each with its own model     
     - **`_parse_inner_prompt()`**: New parser method for parsing `PROMPT ... SELECT ... GENERATE ...` inside CTE       
  parens                                                                                                                
     - **String literal GENERATE instructions**: `GENERATE fn(arg, "instruction with {param} substitution")` —          
  executor extracts the Literal and substitutes `{alias}` placeholders from context                                     
     - **Quoted model names**: All model names must be in double quotes in SPL (`"qwen2.5"`, not `qwen2.5`)             
     - **CTE structure in v2**: CTEs come AFTER `USING MODEL` and BEFORE `SELECT` in the PROMPT statement               
     - **Cache control**: `Executor(cache_enabled=False)` default; `--cache` CLI flag to enable                         
     - **PocketFlow**: 100-line, zero-dependency agentic workflow library — Nodes with `prep/exec/post`, shared         
  store, `>>` operator for connections, conditional branching via `- "action" >>` operator                              
     - **Text2SPL**: LLM-powered translation from natural language to valid SPL syntax, with retry on parse failure     
   (up to 3x)                                                                                                           
     - **SPL-Flow**: Commercial platform on top of SPL engine — Streamlit MVP with PocketFlow orchestration             
     - **human×AI**: Design philosophy — multiplicative not additive collaboration

     # SPL-Flow                                                                                                         
                                                                                                                        
     **Declarative LLM Orchestration — powered by SPL + PocketFlow**                                                    
                                                                                                                        
     SPL-Flow is a Streamlit-based MVP that translates free-form natural language queries into [SPL (Structured         
     Prompt Language)](https://github.com/digital-duck/SPL), routes sub-tasks to specialist language models in          
     parallel, and synthesizes a composed final response — either inline (sync) or saved to file (async).               
                                                                                                                        
     ---                                                                                                                
                                                                                                                        
     ## Architecture                                                                                                    
                                                                                                                        
     ```                                                                                                                
     User Query (free-form text)                                                                                        
             │                                                                                                          
             ▼                                                                                                          
       ┌─────────────┐                                                                                                  
       │  Text2SPL   │  LLM translates NL → SPL syntax                                                                  
       │   Node      │◄── retry on parse failure (up to 3x)                                                             
       └──────┬──────┘                                                                                                  
              │                                                                                                         
              ▼                                                                                                         
       ┌─────────────┐                                                                                                  
       │  Validate   │  SPL parse + semantic analysis                                                                   
       │   Node      │──► retry ──► Text2SPL                                                                            
       └──────┬──────┘                                                                                                  
              │ "execute"                                                                                               
              ▼                                                                                                         
       ┌─────────────┐                                                                                                  
       │   Execute   │  parse → analyze → optimize → run                                                                
       │   Node      │  (parallel CTE dispatch via asyncio)                                                             
       └──────┬──────┘                                                                                                  
              │                                                                                                         
         ┌────┴────┐                                                                                                    
         │         │                                                                                                    
         ▼         ▼                                                                                                    
       Sync      Async                                                                                                  
      Deliver   Deliver                                                                                                 
     (inline)  (/tmp file                                                                                               
                + email*)                                                                                               
     ```                                                                                                                
                                                                                                                        
     *Email delivery: SMTP integration planned for v0.2.

  ---                                                                                                                   
  Technical challenges (the genuinely hard ones):                                                                       
                                                                                                                        
  1. Searchable encryption — The crypto problem nobody has fully solved at scale. If you zero-knowledge encrypt the     
  vault, the server can't search it. Client-side search works but is slow. Searchable Symmetric Encryption exists in    
  research but isn't production-ready. Practical compromise: encrypt document content, keep embeddings in a separate    
  index with a clear privacy disclosure. Users pick their tradeoff.                                                     
  2. Vector store sync — FAISS has no replication protocol. You'd need to build delta sync, conflict resolution, and    
  versioned snapshots. Non-trivial distributed systems work.                                                            
  3. GDPR right to erasure — "Forget me" in a vector index requires re-indexing the entire collection without that      
  document's embedding. Computationally expensive at scale. Needs an architectural answer upfront, not a retrofit.      
  4. Browser-native SPL engine — WASM via Pyodide is heavy (~50MB). The clean path is rewriting the SPL engine core in  
  Rust or Go for a lean WASM target. That's a significant investment.                                                   
  5. Multi-modal RAG retrieval — Text is solved. Searching across audio and image embeddings, cross-modally, is still   
  active research. The ZiNets "find the character image that matches this pronunciation" query is harder than it sounds.
